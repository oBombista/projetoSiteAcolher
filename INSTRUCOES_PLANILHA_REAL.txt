🎯 PROBLEMA RESOLVIDO:
✅ Deploy agora usa planilha real em vez de dados de exemplo
✅ Sistema detecta automaticamente se tem dados reais
✅ Backup automático dos dados de exemplo

🚀 COMO USAR SUA PLANILHA REAL:

📋 MÉTODO MAIS FÁCIL:
1. Clique duas vezes em "preparar_deploy.bat"
2. Escolha opção 1 (Preparar planilha real)
3. Execute os comandos git que aparecerão
4. Pronto! Deploy atualizado com dados reais

📋 MÉTODO MANUAL:
1. Renomeie sua planilha para "Cadastro_Visitantes.xlsx"
2. Execute: python subir_planilha_real.py
3. Escolha opção 1
4. Execute os comandos git

📋 MÉTODO DIRETO:
1. Renomeie sua planilha para "Cadastro_Visitantes.xlsx"
2. git add Cadastro_Visitantes.xlsx
3. git commit -m "Adicionar planilha real"
4. git push origin main

🔍 ARQUIVOS RECONHECIDOS:
✅ Cadastro_Visitantes.xlsx
✅ cadastro_visitantes.xlsx
✅ visitantes.xlsx
✅ dados_visitantes.xlsx

📊 VERIFICAÇÕES AUTOMÁTICAS:
✅ Detecta se tem planilha real
✅ Verifica quantidade de dados (>1 linha)
✅ Confirma estrutura das colunas
✅ Faz backup dos dados de exemplo

💡 MENSAGENS DO SISTEMA:
✅ "Dados reais carregados com sucesso!" = Planilha real funcionando
⚠️ "Planilha com poucos dados" = Poucos registros, usando exemplo
📊 "Planilha não encontrada" = Sem planilha, usando exemplo

🔄 ALTERNAR DADOS:
📊 Voltar aos exemplos: python subir_planilha_real.py (opção 2)
📋 Usar planilha real: python subir_planilha_real.py (opção 1)

🛠️ PROBLEMAS COMUNS:
❌ "Nenhuma planilha encontrada" → Renomeie para Cadastro_Visitantes.xlsx
❌ "Poucos dados" → Verifique se tem registros reais
❌ "Erro no git" → Verifique permissões do repositório
